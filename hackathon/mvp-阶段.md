# MVP 阶段规划

> 来源：Obsidian `raw/docs/zh/part4/project-guide.md`（项目实战指南）

## 阶段 1：单机演示版

**目标：** 证明核心价值，不追求完整产品闭环。

- [ ] 固定一份样例输入（提案文本 / 地址 / 协议文档链接）
- [ ] AI 处理流程可在本地跑通（脚本或 Notebook）
- [ ] 输出为结构化摘要或风险要点（无需上链）

**产出路径：** `experiments/阶段1-单机演示/`

## 阶段 2：可交互原型

**目标：** 用户能操作输入并看到输出。

- [ ] 最小 UI 或 CLI 交互
- [ ] 输入校验与错误提示
- [ ] 输出带来源引用（RAG / 链接）

**产出路径：** `experiments/阶段2-可交互原型/`

## 阶段 3：接入真实链上数据

**目标：** 只读链状态进入上下文（Chain-aware Context）。

- [ ] RPC / 索引数据只读查询
- [ ] 链 ID、地址、合约校验
- [ ] 不在此阶段开放自动转账

**产出路径：** `experiments/阶段3-链上只读/`

## 阶段 4：有限执行或持续监控

**目标：** 仅在边界清楚时加入执行或告警。

- [ ] Session Key / 策略 / 人工确认之一
- [ ] 金额与合约白名单
- [ ] 审计日志与撤销路径

**产出路径：** `experiments/阶段4-有限执行/`（黑客松前完成其一即可）
