# Planning（规划）

> Agent 将目标拆解为步骤的候选路线；计划用于暴露依赖与风险，**不是**对执行的授权。

## 解释

Planning 输出「先读提案 → 再查投票权 → 再列风险」等序列。好的计划标明每步工具、读写属性、可否自动、失败是否可重试、完成如何验证。模型计划会随上下文变化，须允许用户修改或否决；高风险步骤应插入人审节点。

与 [[concepts/agent]] 的 state 配合：计划版本、当前 step、已完成工具结果应外置存储。

## 例子

- **可审计划 UI**：展示 5 步计划，用户取消「自动提交 vote」步骤。
- **条件分支**：若 `simulation.revert` 则停止并改计划为「仅解释错误」。
- **预算上限**：计划预估 token/工具调用次数，超限则缩短或请求确认。

## 误区

- **「有计划就能自动跑完」**：计划不等于授权。
- **「计划越细越安全」**：过长计划可能引入多余写步骤。
- **「模型自我修正一定可靠」**：Reflection 仍可能巩固错误假设。

## 使用边界

| 适合 | 不适合 |
|------|--------|
| 暴露步骤供人审 | 替代权限与模拟检查 |
| 与 eval 测计划合理性 | 无停止条件的无限重规划 |
